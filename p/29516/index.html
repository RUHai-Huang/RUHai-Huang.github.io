前言

前人栽树，后人乘凉！
本网站在实验室H师兄的帮助和启发下勉强完成的，因此想重新复盘和记录下心路历程，其中绝大部分内容搬运和参考了H师兄的博客royhuang’s blog以及主题Hexo-Butterfly官方文档，最后再次感谢师兄对本网站的鼎力相助。
本网站的初衷是（装逼）熟悉和学习建站的流程（其实并没有什么难度，框架的集成度已经非常高了）。自己平时学习知识的时候也喜欢在语雀上记笔记，因此会不定期将学习笔记整理归纳到本网站中。

主流建站方式比较

建站难度 所需知识 定制化程度 适合人群 备注
WordPress 🌟🌟 Linux、Git基本知识 一般 Web开发小白 WordPress从0开始建站
Java Web 🌟🌟🌟🌟 Web前后端开发知识，多且杂 非常高，前后端代码全栈开发 Java Web开发人员
Hexo + Github Pages 🌟🌟 Linux、Git基本知识 较高，基于配置文件修改较方便 Web开发小白 不需要备案域名
Hexo + 腾讯云 🌟🌟 Linux、Git基本知识 较高，基于配置文件修改较方便 Web开发小白 域名需要备案，但网站访问速度会变快很多
  • Java Web全栈搭建？
    • 适合Java Web为技术栈的专业人员，可以作为个小项目；
    • 但如果你并非专业的Java Web开发人员，不想学习复杂的Java Web体系，但是又具备开发基础知识，想搭建一个漂亮的博客，WordPress 和 Hexo是你不二的选择。
  • WordPress还是Hexo？
    • WordPress优点：很简单，适合特别小白的同学；
    • WordPress缺点：占用资源多，响应慢；不支持原生markdown（每次要导出.html太痛苦了）；主题也不够美观。

所以还是推荐使用Hexo。

  • Hexo + Github Pages 还是 Hexo + 腾讯云?
    • 如果使用Github Pages 托管网站，因为服务器在国外（Github），域名无需备案，但访问速度可能会慢（自我体验还可接受），也无需购买云服务器；
    • 如果使用腾讯云托管网站，需要购买云服务器（或COS），且域名需要备案，但访问速度会很快。

​

本文考虑更低成本，也不想备案，访问速度虽然慢点但还能接受，所以采用Hexo + Github Pages作为最终的建站方案。

  1. TIPS

💡 一般我们发布文章（或修改主题等）, 流程如下：

  • **hexo clean && hexo g && hexo s** ，打开 localhost:4000本地预览；
  • 本地进行修改（文章增改、CSS样式修改）；F5刷新下网页即可看到修改后的文章或样式，一般不需要重新执行以上三个命令；
  • 但是如果修改了配置文件则需要以上三个命令；
  • 将本地修改同步到GitHub上**hexo clean && hexo g && hexo d**。
